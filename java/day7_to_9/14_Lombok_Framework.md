# Lombok Framework

**Project Lombok** is a Java library that automatically plugs into your editor and build tools, spicing up your Java. It is designed to **reduce boilerplate code** (repetitive code like getters, setters, constructors, `toString`, `equals`, etc.).

## 1. How It Works
Lombok uses **Annotations** to generate code during compilation.
-   You write the annotation (e.g., `@Data`).
-   Lombok processes the source file.
-   The generated bytecode contains all the methods as if you wrote them manually.
-   This keeps your source code clean and readable.

## 2. Installation Steps
1.  **Download**: Get `lombok.jar` from [projectlombok.org](https://projectlombok.org/download).
2.  **IDE Integration**:
    -   Double-click `lombok.jar` to run the installer.
    -   Select your IDE (e.g., Eclipse execution file).
    -   Click **Install/Update**.
    -   Restart IDE.
3.  **Project Setup**:
    -   Add `lombok.jar` to your project's Build Path (Classpath) or Maven dependencies.

## 3. Key Annotations

### A. @Data
This is the most popular annotation. It is a shortcut that bundles:
-   `@ToString`
-   `@EqualsAndHashCode`
-   `@Getter` (for all fields)
-   `@Setter` (for all non-final fields)
-   `@RequiredArgsConstructor`

### B. Constructors
-   `@NoArgsConstructor`: Generates a no-argument constructor.
-   `@AllArgsConstructor`: Generates a constructor with one parameter for every field in your class.

#### Code Example

```java
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

@Data                // Generates Setters, Getters, toString, equals, hashCode
@NoArgsConstructor   // Generates public MyClass() {}
@AllArgsConstructor  // Generates public MyClass(int num, String name) {}
public class MyClass {
    private int num;
    private String name;
}

// --- Usage ---
public class Demo {
    public static void main(String args[]) {
        // Constructor usage (generated by @AllArgsConstructor)
        MyClass m1 = new MyClass(100, "Scott");
        
        // Setter usage (generated by @Data)
        m1.setNum(200);
        
        // Getter usage (generated by @Data)
        System.out.println(m1.getNum()); 
        
        // toString usage (generated by @Data)
        System.out.println(m1); // Output: MyClass(num=200, name=Scott)
    }
}
```

## 4. Benefits
1.  **Reduced Code**: A 50-line class usually shrinks to 5-10 lines.
2.  **Readability**: Developers focus on fields (data logic) rather than scrolling through trivial methods.
3.  **Maintainability**: No need to manually update `toString` or constructors when you add a new field; Lombok handles it automatically.
